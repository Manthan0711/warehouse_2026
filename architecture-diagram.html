📊 Complete System Data Flows

Owner Listing Flow: Owner (Frontend) → Forms Input → Warehouse Service → PostgreSQL Database → Storage (Images) → Admin Approval → Live to Seekers
LLM-Powered Search: Seeker Query → React Frontend → OpenAI Intent Extraction → Search Service → PostgreSQL Query → Storage URLs → Display Results
Booking Workflow: Seeker Selection → Check Availability → Create Booking → Database → Real-time Notification → Owner Approval
Review & AI Analysis: Submit Review → Sentiment Analysis (OpenAI) → Store in DB → Calculate Rating → Display on Listing

🔑 Key Architecture Concepts

  • Monolithic Backend: All backend logic runs through Supabase REST API (no separate microservices)
  • JWT Authentication: Every request includes JWT token verified against user roles
  • Row Level Security: PostgreSQL RLS policies enforce authorization at database level
  • Real-time Subscriptions: WebSocket connections deliver instant updates to frontend
  • OpenAI Integration: Natural language processing for search and sentiment analysis
  • Stateless Frontend: React SPA with no server-side sessions; state managed in JWT
